# 1. Executive Concept

**What CraftlyCV Is:**
CraftlyCV is an AI-powered application engine that transforms one master career profile into multiple ATS-safe, job-specific resumes and application assets. It is not a resume template builder—it is a career application operating system built for specialized professionals who have historically been poorly served by generic AI tools.

**Core Thesis:**
Specialized professionals (Cybersecurity, Nursing, Skilled Trades, Creative Tech) lose interviews not because they lack credentials, but because their applications get filtered by ATS before human eyes see them. CraftlyCV solves this with niche-intelligent rewriting, match scoring, proof gap detection, and one-click tailoring—delivered at a one-time price point that respects the reality of short job-search windows.

**Primary Value Proposition:**
"Upload once. Tailor for every role. Apply with confidence."

**Brand Promise:**
CraftlyCV helps specialized job seekers convert one master career profile into ATS-safe, job-specific resumes and application assets in minutes—not hours.

**Strategic Enemy (not competitors):**
- Generic AI output that flattens specialized experience
- ATS uncertainty and auto-rejection anxiety
- Expensive monthly subscriptions for short job searches
- Repetitive manual tailoring for each application
- Broad "template builder" products designed for generic careers

**Category Position:**
Position against Enhancv, Rezi, and Teal by being niche-first where they are broad, outcome-focused where they are editor-focused, and one-time-priced where they are subscription-modeled.

# 2. Market Need Analysis

## Market Context

The global resume builder market is ~$3.2B, growing at 6.8% CAGR. However, virtually all major players (Enhancv, Rezi, Teal, Kickresume, Resume.io) target generic professionals with generic AI. This creates a structural gap:

**Problem 1: Generic AI Fails Specialized Professionals**
A cybersecurity engineer with CISSP, Security+, incident response experience, SIEM tooling, and cloud security credentials gets the same AI suggestions as a marketing manager. The output is generic, diluted, and ATS-weak.

**Problem 2: Subscription Fatigue**
Job seekers face 3–6 month search windows. Paying $20–$50/month for a resume tool feels punitive, especially when the purchase is sporadic and time-bounded. One-time pricing aligns with the actual purchase behavior.

**Problem 3: ATS Anxiety Is Real**
62% of recruiters report their ATS filters out qualified candidates. Job seekers have no visibility into why. They need confidence signals, not just templates.

**Problem 4: Niche Terminology Is a Moat**
Cybersecurity roles require specific frameworks (NIST, MITRE ATT&CK), tools (Splunk, CrowdStrike, Qualys), and certifications (CISSP, CISM, OSCP). Nursing requires license numbers, specializations, patient ratios, EMR fluency. These are not resume fields—they are credibility signals that generic builders don't understand.

**Problem 5: Proof Gaps Kill Applications**
A candidate might have 10 years of experience but not articulate it in ways that map to job descriptions. Missing proof signals (metrics, scope, impact) create match score drops that candidates never see.

   - 500K+ professionals
   - $90K–$200K average salary
   - Portfolio and project demonstration needs
   - Dark-mode, modern aesthetic preferences
   - GitHub/stack/technology fluency required

**Total Initial Addressable Market: ~17M professionals**
**Serviceable Obtainable Market (Year 1): 50K–100K users**

# 3. User Segments and Jobs-to-be-Done

## Primary User Segments

### Segment 1: The Active Job Seeker (Primary)
**Profile:** Currently applying, 2–8 weeks into active search, experiencing ATS rejections, spending hours tailoring each application.

**Emotional State:** Anxious, time-pressured, skeptical of generic tools, wants proof that their next application is stronger.

**JTBD:**
- Import existing resume or LinkedIn text quickly
- Paste a job description and see immediate match insights
- Generate a tailored version in under 5 minutes
- Understand what is missing before submitting
- Feel confident the resume won't be auto-filtered

**Willingness to Pay:** High—if the value is clear and immediate. One-time pricing preferred over subscriptions for this segment.

### Segment 2: The Career Transitioner (Secondary)
**Profile:** Moving from one specialty to another (e.g., IT admin → Cybersecurity, CNA → RN). Needs to reframe existing experience for new terminology.

**Emotional State:** Uncertain about how to translate credentials, worried about appearing underqualified, needs guidance on missing skills.

**JTBD:**
- Identify which existing skills transfer
- Understand what credentials are missing
- Generate application language that bridges the gap
- See proof of relevance to the new niche

### Segment 3: The Passive Explorer (Tertiary)
**Profile:** Not actively applying but wants to be ready. Exploring options, benchmarking their profile against job descriptions.

**Emotional State:** Curious, aspirational, not yet urgency-driven.

**JTBD:**
- See a match score for their current resume against ideal roles
- Understand proof gaps without committing to a full tailoring session
- Get a premium visual preview to motivate action

**Monetization:** Lower priority. Can be converted to Active Seeker via urgency triggers.

# 6. Complete Feature Architecture

## A. Authentication and Account System

### Auth Methods
- Email/password sign up with strong password requirements (min 8 chars, 1 uppercase, 1 number)
- Google OAuth login
- Magic link login (passwordless email)
- Email verification required before full access
- Forgot password flow with secure reset link
- Session management with 30-day inactivity timeout
- Logout everywhere (revoke all sessions)
- Device/session list view in settings
- Optional 2FA for premium users (TOTP)
- Captcha on signup/login (reCAPTCHA v3 or hCaptcha)
- Rate limiting: 5 failed login attempts → 15 min lockout, 10 signups per IP/day

### Security Layer
- Argon2id password hashing
- CSRF tokens on all state-changing requests
- Secure, HttpOnly, SameSite=Strict cookies
- JWT with short expiry (15 min) + refresh tokens (30 days)
- Session rotation on privilege elevation
- All inputs server-side validated
- File uploads: PDF/DOCX/TXT only, max 10MB, validated MIME type
- Audit log for account actions (login, logout, password change, data export, data deletion)

### Compliance
- Terms of service consent at signup (checkbox, immutable record)
- Privacy policy consent at signup (checkbox, immutable record)
- Privacy policy accessible from every page footer
- Data export (GDPR/CCPA): downloadable JSON of all user data
- Account deletion: confirmation flow, 24-hour grace period, full data purge
- AI safety: every generated claim flagged as "unverified—review before export"

# 9. Authentication and Security System

## Authentication Flow

### Signup
1. User enters email + password (or clicks Google OAuth)
2. Client-side validation (format, strength)
3. Server-side validation
4. Password hashed with Argon2id
5. User record created (email, hashed password, plan: free)
6. Email verification token generated
7. Verification email sent
8. User shown "Check your email" screen
9. On email verification click → activate account → redirect to onboarding

### Login
1. User enters email + password
2. Rate limit check (5 failures → 15 min lockout)
3. Validate credentials
4. Generate JWT (15 min expiry) + refresh token (30 days)
5. Set secure, HttpOnly cookie with refresh token
6. Return JWT in response
7. Redirect to dashboard

### Magic Link
1. User enters email
2. Server generates one-time login token (15 min expiry)
3. Email sent with magic link
4. User clicks link → validate token → create session → redirect

### Password Reset
1. User enters email
2. If exists, send reset email with token (1 hour expiry)
3. User clicks link → enter new password
4. Validate token, hash new password, update record
5. Invalidate all existing sessions
6. Redirect to login with success message

## Session Management

- JWT access tokens (15 min expiry)
- Refresh tokens stored in HttpOnly cookie (30 days)
- Refresh token rotation on each use
- All sessions revocable ("Logout everywhere")
- Session list shows: Device, Location (approx), Last active
- "Revoke" button per session
- Inactivity timeout: 30 days

## Two-Factor Authentication (Optional, Premium)

- TOTP-based (Google Authenticator, Authy)
- Setup: Show QR code, backup codes
- Required on: Login, password change, data export, account deletion
- Backup codes: 10 single-use, shown once on setup

## Rate Limiting

| Endpoint | Limit |
|----------|-------|
| POST /login | 5 attempts per 15 min |
| POST /signup | 10 per IP per day |
| POST /forgot-password | 3 per email per hour |
| POST /analyze | 10 per user per hour |
| POST /ghost-write | 5 per user per hour |
| General API | 100 requests per minute |

## File Upload Security

- Allowed types: PDF, DOCX, TXT
- Max size: 10MB
- Server-side MIME type validation
- Magic byte verification for PDF/DOCX
- Virus scan if file size < 5MB (ClamAV or similar)
- Uploaded files stored in private S3 bucket (not public)
- Files only accessible via authenticated presigned URLs
- Auto-delete uploaded files after 7 days (or on user deletion)

## AI Safety Layer

- No model should suggest unverified credentials
- No model should fabricate metrics
- All generated claims flagged: "Verify before export"
- Prompt injection defenses on JD input
- Output filtering for disallowed content
- Confidence scores on all suggestions
- "AI suggestions are editable and reviewable before export" visible everywhere

## Audit Logging

Log all account actions:
- Login / Logout (each session)
- Password change
- Email change
- 2FA enable/disable
- Data export request
- Data deletion request
- Plan upgrade
- API key creation/revocation

Log format: `{user_id, action, ip, user_agent, timestamp, metadata}`

## Data Privacy Controls

- "Your resume data stays private" — visible on dashboard
- "We never sell your career data" — visible on signup
- "Delete your data anytime" — in settings
- GDPR/CCPA compliant
- Data export: One-click JSON download
- Data deletion: Confirmation → 24hr grace period → full purge
- Privacy policy: Plain-language summary, not just legalese

## AI Workflow

### Ghost-Writer Pipeline
1. User submits: vault_data + job_description + niche + seniority
2. Server validates and sanitizes inputs
3. Server calls Gemini API with structured prompt
4. Prompt includes:
   - Full master resume data
   - Target job description
   - Niche context (skill taxonomy, bullet prompts)
   - Output format specification (JSON)
   - Safety instructions
5. AI returns structured JSON
6. Server validates response schema
7. Server calculates match score
8. Server calculates ATS risk score
9. Server saves tailored version
10. Server returns result to client

### Match Score Algorithm
```
match_score = (
  keyword_overlap * 0.4 +
  skill_match * 0.3 +
  proof_signal_density * 0.2 +
  seniority_alignment * 0.1
) * 100
```

### ATS Risk Score
```
ats_risk = (
  table_usage * 20 +
  graphic_elements * 15 +
  sidebar_presence * 15 +
  non_standard_headings * 10 +
  keyword_stuffing * 20 +
  formatting_inconsistency * 20
)
-- Inverted: lower risk is better
-- Display as: "ATS Safety Score: 85/100"
```

## Security Implementation

### Password Hashing
- Argon2id via `argon2` Node.js library
- Memory: 64MB, Iterations: 3, Parallelism: 4

### JWT Strategy
- Access token: 15 min expiry, stored in memory (not localStorage)
- Refresh token: 30 days, HttpOnly cookie
- Token rotation on refresh

### Input Validation
- Zod schemas for all API inputs
- Server-side validation on every endpoint
- JD input: Max 50,000 characters
- No HTML allowed in text inputs (XSS prevention)

### File Upload Validation
- Validate MIME type from magic bytes
- Scan with ClamAV (if available) for files < 5MB
- Store in private S3 bucket
- Presigned URLs for access (15 min expiry)
